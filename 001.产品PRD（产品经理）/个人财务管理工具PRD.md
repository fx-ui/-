## 1. 产品概述

### 1.1 产品定位

**每日记账**是一款面向个人及小微个体的轻量级财务管理工具，V1.0 以**移动端 H5 网页**形态交付（适配手机浏览器访问）。用户可以每天快速记录收入与支出，并通过可视化的统计分析了解自己的消费习惯，从而做出更理性的财务决策。

**V1.0 产品形态**：移动端 H5 网页（PWA 可添加到桌面），无需下载安装，浏览器即开即用。后续版本逐步覆盖 Android/iOS 原生 App、微信/支付宝小程序。

### 1.2 核心价值主张

- **快**：3 秒完成一笔记账，降低记录门槛
- **清**：图表直观呈现钱花在哪里，一目了然
- **省**：通过预算提醒和趋势分析，帮助用户节省开支
- **轻**：H5 网页形态，无需下载安装，浏览器输入网址即开即用

### 1.3 产品愿景

帮助 100 万用户养成记账习惯，实现"每一笔都有数，每一月都清楚"的财务生活。

## 2. 目标用户

### 2.1 用户画像

| 画像 | 年龄段 | 描述 | 核心诉求 |
|------|--------|------|---------|
| **大学生** | 18-22 岁 | 生活费有限，社交和购物支出多 | 控制生活成本、AA 记账、简单好用 |
| **职场新人**（主力） | 22-28 岁 | 刚参加工作，收入有限但消费场景多 | 想知道钱花哪了、控制月光、攒钱目标 |
| **职场中层/家庭** | 28-45 岁 | 有家庭开支、房贷/车贷等固定支出 | 分类管理、家庭预算、导出报表 |
| **中老年个人** | 45 岁以上 | 退休或临近退休，关注养老金和日常开销 | 字体大号、操作极简、收支一目了然 |
| **小微商户** | 不限 | 个体户、自由职业者、小店主，需简单记账 | 经营收支分开、简易流水、收入分类统计 |
| **自由职业者** | 不限 | 摄影师、设计师、咨询师等，收入不固定 | 多收入来源管理、支出控制、税务简易对账 |

### 2.2 使用场景

#### 个人场景

1. **日常消费即时记录**：买了一杯咖啡、吃了一顿饭，付款后立刻掏出手机打开网页记一笔（10 秒内完成）
2. **晚间回顾补记**：晚上躺床上回顾一天花了多少钱，批量补记（2 分钟内完成当天记录）
3. **月底对账**：月底发工资后想看看本月花了多少、还剩多少、钱都去哪了
4. **大额消费决策前**：想买个东西，先看看本月预算还剩多少

#### 小微经营场景

5. **小店日常流水**：个体小店主记录每日进货支出和销售收入，月底看净收入
6. **自由职业项目管理**：自由设计师按项目/客户分类记录收入和支出，年底方便核算个税
7. **家庭+生意账本分离**：通过多账户切换，在同一工具中分开管理家庭开支和生意流水

## 5. 功能详细说明

### 5.1 记账模块

#### 5.1.1 快速记账（核心功能）

**优先级**：P0（必须有）

**功能描述**：
用户在主界面即可快速完成一笔支出/收入记录，无需跳转多个页面。

**输入字段**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 金额 | 数字 | ✅ | 支持小数点后两位，最大 999,999,999.99 |
| 类型 | 单选 | ✅ | 支出 / 收入，默认支出 |
| 分类 | 选择 | ✅ | 预置分类 + 自定义分类 |
| 日期 | 日期 | ✅ | 默认当天，可修改 |
| 账户 | 选择 | — | 从哪个账户支出/收入 |
| 备注 | 文本 | — | 最多 200 字 |

**预置分类体系**：

##### 个人支出分类

| 大类 | 子类 |
|------|------|
| 🍽️ 餐饮 | 三餐、零食、水果、饮品、聚餐 |
| 🚗 交通 | 公交地铁、打车、加油、停车 |
| 🛒 购物 | 服饰、日用品、数码、家居 |
| 🏠 居住 | 房租、水电、物业、维修 |
| 🎮 娱乐 | 电影、游戏、运动、旅游 |
| 💊 医疗 | 看病、买药、体检 |
| 📚 学习 | 书籍、课程、培训 |
| 👫 社交 | 送礼、红包、聚会 |

##### 个人收入分类

| 大类 | 子类 |
|------|------|
| 💰 薪资 | 工资、奖金、公积金提取 |
| 💼 兼职 | 兼职收入、外包项目、稿费 |
| 📈 投资 | 基金、股票、理财收益、利息 |
| 🔄 退款 | 购物退款、报销 |

##### 小微经营分类

| 大类 | 支出子类 | 收入子类 |
|------|---------|---------|
| 📦 进货/采购 | 原材料、商品采购、包装物料 | — |
| 🏬 店铺运营 | 租金、水电、物业、设备维护 | — |
| 📣 营销推广 | 广告投放、促销物料、推广费用 | — |
| 👷 人力 | 兼职工资、外包费用 | — |
| 💼 经营收入 | — | 销售收入、服务费、项目款、预收款 |
| 📋 行政/税费 | 营业执照、税务申报、社保 | 政府补贴 |
| 🔧 经营其他 | 快递物流、交通差旅、自定义 | 自定义 |

**交互设计要点**：
- 打开网页即进入记账界面，焦点自动落在金额输入框
- 金额输入使用大号数字键盘，支持小数点快捷输入
- 常用分类置顶显示（基于使用频率动态排序）
- 备注字段支持语音输入转文字（Web Speech API）
- 保存成功后给予轻反馈（Toast 提示 + 过渡动画），不阻断后续操作

#### 5.1.2 模板记账

**优先级**：P1（应该有）

**功能描述**：
对于重复性消费（如每天买咖啡、每月交房租），用户可创建记账模板，一键完成记录。

**模板字段**：
- 模板名称（如"早晨咖啡"）
- 预填金额、分类、账户
- 快捷备注（如"星巴克拿铁"）

#### 5.1.3 账单列表与编辑

**优先级**：P0（必须有）

**功能描述**：
- 按日期倒序展示所有账单流水
- 每条记录展示：分类图标、金额、备注摘要、账户
- 左滑删除（支持触控滑动），点击进入编辑
- 支持按日期、分类、金额范围筛选
- 支持关键词搜索（搜备注、分类名）

### 5.2 统计分析模块

#### 5.2.1 支出分类饼图

**优先级**：P1（应该有）

**功能描述**：
- 以饼图/环形图展示选定时间范围内的支出构成
- 图例展示各分类占比和金额
- 点击饼图某个扇区可钻取查看该分类下的详细记录

#### 5.2.2 月度收支趋势图

**优先级**：P1（应该有）

**功能描述**：
- 柱状图+折线图组合展示月度收支趋势
- 柱状图：每月总支出 vs 总收入
- 折线图：每月结余趋势
- 时间范围：近 3 个月 / 6 个月 / 12 个月

#### 5.2.3 周报 / 月报 / 年报

**优先级**：P2（可以有）

**功能描述**：
- 自动生成周期消费报告
- 周报：每周日晚 8 点通过浏览器通知（Web Notification API）推送摘要
- 月报：每月 1 日推送
- 内容包括：总支出、与上期对比、最大支出分类、异常大额消费提醒
- 用户也可在统计页主动查看各期报告

### 5.3 预算管理模块

#### 5.3.1 预算设置

**优先级**：P1（应该有）

**功能描述**：
- 设置月度总预算（如每月消费不超过 5000 元）
- 支持设置分类预算（如餐饮 ≤ 1500 元）
- 预算周期：月 / 周
- 支持预算重置提醒

#### 5.3.2 预算预警

**优先级**：P1（应该有）

**功能描述**：
- 进度条展示：已花 / 预算总额
- 预警节点：
  - 绿色（< 60%）：正常
  - 黄色（60% - 80%）：提醒
  - 橙色（80% - 100%）：警告
  - 红色（≥ 100%）：超预算
- 超预算时推送通知
- 在记账页面实时展示当日/当月剩余预算

| 🏦 银行卡 | 工资卡、储蓄卡、信用卡 |

##### 小微经营账户

| 类型 | 示例 |
|------|------|

| 📱 经营收款 | 商户版支付宝、微信商户 |

#### 5.4.2 信用卡管理

**优先级**：P3（后期版本）

**功能描述**：

### 5.5 数据管理模块

#### 5.5.1 数据导出

**优先级**：P2（可以有）

**功能描述**：
- 支持导出为 CSV / Excel 格式
- 可选择导出时间范围
- 导出内容包括：日期、类型、分类、金额、账户、备注

#### 5.5.2 数据备份与恢复

**优先级**：P2（可以有，V1.2 重点实现）

**功能描述**：
- **V1.0 策略**：数据存储在浏览器 IndexedDB 中，支持手动导出 JSON 文件到手机本地

- 引导用户定期导出数据文件手动备份
- PWA 持久化存储不受浏览器缓存清理策略影响

## 8. 非功能性需求

### 8.1 性能要求

| 指标 | 要求 |
|------|------|
| 首屏加载时间（LCP） | ≤ 2.0 秒（4G 网络下） |
| 页面切换响应 | ≤ 300ms |
| 记账保存响应 | ≤ 200ms（localStorage / IndexedDB 本地存储） |
| 图表渲染时间 | ≤ 500ms（1000 条数据以内） |
| 首屏 JS 体积 | ≤ 200KB（gzip 压缩后） |
| PWA 离线可用 | 核心功能（记账、查账）完全离线可用 |

### 8.2 平台要求

- **V1.0 产品形态**：移动端 H5 网页（SPA 单页应用）
- **目标浏览器**：iOS Safari 14+ / Android Chrome 90+ / 微信内置浏览器
- **屏幕适配**：响应式设计，适配 320px - 428px 宽度（覆盖主流手机屏幕）
- **PWA 支持**：支持添加到手机桌面，离线缓存核心页面
- **数据存储**：浏览器端优先（IndexedDB），支持登录后云端同步（可选）
- **后端技术栈**：建议 Node.js / Go 轻量 API + MySQL / PostgreSQL
- **服务部署**：云服务器 + CDN 加速静态资源，国内用户优先

### 8.3 安全与隐私

- 全站 HTTPS，HSTS 强制加密传输
- 浏览器端数据使用 Web Crypto API（AES-GCM）加密存储
- 云端同步数据全程加密传输，数据库字段级加密
- 不上传用户地理位置、通讯录等隐私信息
- 用户注销后 30 天内彻底清除云端数据
- Cookie/Token 设置 HttpOnly + Secure + SameSite=Strict
- CSP（内容安全策略）防止 XSS 攻击

### 8.4 设计原则

1. **极简优先**：每个页面只做一件事，减少认知负荷
2. **即时反馈**：每次操作都有明确的视觉/触觉反馈（Toast 提示、过渡动画）
3. **容错设计**：记账可撤销（3 秒内撤销）、删除需二次确认
4. **渐进披露**：高级功能（预算、报表）不干扰基础记账体验
5. **离线可用**：核心记账功能基于 Service Worker 离线缓存，断网也能记账
6. **拇指操作区**：核心交互元素（金额输入、分类选择、保存按钮）放在屏幕下半部拇指热区
7. **触控友好**：按钮最小点击区域 44×44px，列表项高度 ≥ 48px，防误触
8. **大字适配**：支持字体大小调节，照顾中老年用户阅读体验

### 8.5 视觉设计规范

#### 8.5.1 设计风格定位

**关键词**：粉嫩 · 可爱 · 温暖 · 治愈 · 轻量

整体视觉像一本「软萌手账本」——圆润的卡片、柔和的粉彩色块、俏皮的小图标，让记账这件事从"令人焦虑的数字任务"变成"温馨治愈的每日小仪式"。

## 10. 风险与假设

### 10.1 关键假设

| 序号 | 假设 | 验证方式 |
|------|------|---------|
| 1 | 用户有记账意愿但被复杂操作劝退 | 竞品分析 + 用户访谈 |
| 2 | 3 秒快速记账能显著提升记账频率 | A/B 测试不同记账流程 |
| 3 | 可视化图表能帮助用户认知消费问题 | 用户行为数据 + NPS 调研 |
| 4 | 预算预警能有效帮助用户控制支出 | 对比设置预算前后的月均消费 |

### 10.2 主要风险

| 风险 | 影响 | 缓解策略 |
|------|------|---------|
| 用户新鲜感过后流失 | 留存率低 | V1.0 即包含周报推送（浏览器通知），保持触达 |
| 竞品功能丰富，用户迁移成本高 | 获客困难 | 聚焦"快"的核心差异化，支持数据导入 |
| 财务数据隐私顾虑 | 信任度低 | 本地优先策略 + 隐私政策透明 + 加密算法公开 |
| 记账分类不够智能 | 操作繁琐 | 后续版本加入 AI 自动分类 |
| Web 形态被浏览器清除缓存 | 数据丢失 | PWA 持久化存储 + 引导用户注册云端备份 |
| 手机浏览器兼容性差异大 | 部分用户无法使用 | 覆盖 Top 5 移动浏览器，渐进增强降级方案 |
| 微信等 App 内嵌浏览器限制 | 功能受限 | 专项适配微信内置浏览器，引导外部浏览器打开 |
| 小微用户需求差异大 | 功能不够用或太复杂 | 新用户引导时选择角色（个人/小微），功能按角色展示 |
| 中老年用户学习门槛 | 不会用 | 大字模式 + 首次使用分步引导 + 减少专业术语 |

### 10.3 竞品参考

| 竞品 | 优势 | 劣势 | 我们的差异化 |
|------|------|------|-------------|
| 随手记 | 功能全面、历史悠久 | 界面复杂、新手门槛高 | 极简设计、3 秒记账 |
| 挖财 | 社区活跃、理财内容多 | 广告多、记账流程不顺畅 | 纯净体验、无广告干扰 |
| MoneyWiz | 专业强大、报表丰富 | 付费订阅、学习成本高 | 免费核心功能、轻量易用 |
| 鲨鱼记账 | 操作简单、UI 美观 | 统计分析浅、缺少预算 | 更全面的预算与报表 |
